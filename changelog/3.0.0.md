# AIMathTutor Changelog — Version 3.0.0

## Added

- Retry mechanism for Gemini and OpenAI AI tutor responses, improving reliability when a provider temporarily fails
- Server-side limits on AI temperature and response length to prevent accidental misconfiguration
- Connection timeouts for all external AI providers to avoid hanging requests
- Comment moderation tracking in the admin panel, showing when and why a comment was moderated and by whom
- Admin AI configuration panel now includes a "Test Connection" button for each provider to verify reachability
- Admin AI configuration panel now supports resetting all settings to their factory defaults with one click

## Changed

- Exercise difficulty levels are now standardized to Beginner, Intermediate, Advanced, and Expert for consistent filtering and display
- Faster exercise loading and comment listing thanks to database query optimizations
- More consistent and helpful error messages across admin interfaces
- Improved AI tutor feedback that references the original problem and expected solution for clearer guidance
- Stricter input validation on user profiles, exercises, groups, and settings to catch mistakes earlier
- AI chat no longer performs unnecessary analysis after a problem is already solved
- Gemini, OpenAI, and Ollama now all use the same default response length (2000 tokens)
- Default OpenAI model updated to gpt-5-nano
- Default Gemini model updated to gemma-3-27b-it
- Vaadin framework updated to 25.1.4

## Fixed

- Duplicate congratulatory notifications no longer appear after undo or redo actions in the math workspace
- Exercise search and date range filtering now work correctly in admin views
- Inequality problem generation no longer truncates fractional solutions
- Comment authorization, user identity checks, and session metrics are now more robust
- Lessons that contain no exercises now display properly
- Loading a new custom problem after solving a previous one now resets the completion state correctly
- Missing timestamps are now recorded on AI interaction history entries
- Raw technical error messages are replaced with user-friendly notifications

## Security

- Passwords must now be at least 8 characters and meet complexity requirements
- Repeated failed login attempts are slowed down with exponential backoff
- Prompt injection protection added via server-side delimiter and length limits on AI requests
- Rate limiting on AI tutor requests to prevent abuse
- Plaintext passwords are no longer kept in active sessions
- Active users are re-verified on every permission check
- Safer AI response parsing and JSON validation
- Admin pages restricted to verified admin users only
- Cross-site scripting (XSS) protections added to exercise content, math workspace, and comment tooltips
- AI provider API keys are now sent in secure headers instead of URLs
- Unsafe external URLs and local network addresses are blocked in AI provider configuration
