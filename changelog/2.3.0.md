# AIMathTutor Changelog — Version 2.3.0

## Added

- Added retry mechanism to Gemini and OpenAI services to match Ollama fault-tolerance patterns
- Enforced server-side hard caps on AI temperature (0.0-2.0) and max-tokens (1-8192)
- Enforced connection timeouts on external AI service calls

## Changed

- Improved overall application performance with database query optimizations and indexing
- Improved error handling in authentication, user management, data and AI services
- Improved AI tutor responses by including original problem state and target solution for better guidance
- Improved input validation across user profiles, exercises, groups, and settings
- Optimized AI tutor by skipping unnecessary action analysis after a problem is solved
- Removed full prompt content and PII from INFO-level logs
- Default OpenAI model updated to `gpt-5-nano`
- Default Gemini model updated to `gemma-3-27b-it`
- Vaadin version updated from 25.1.3 to 25.1.4

## Fixed

- Fixed duplicate congratulatory notifications triggered by undo and redo actions
- Fixed exercise search and date range filtering
- Fixed integer division bug in inequality problem generation that truncated fractional solutions
- Fixed null safety issues in comment authorization, user identity checks, and session metrics
- Populated missing audit fields on AI interactions
- Aligned Gemini HTTP client connect timeout with other AI providers (10 seconds)
- Replaced user-facing raw exception messages with proper error notifications
- Fixed missing lesson cards for lessons that contain no exercises
- Fixed stale completion state when loading a custom problem after solving a previous one

## Security

- Strengthened password requirements to a minimum of 8 characters with complexity rules
- Added exponential backoff throttling for repeated failed login attempts
- Added server-side prompt delimiters and length limits to prevent prompt injection
- Added rate limiting on AI tutor requests to prevent abuse
- Removed plaintext passwords from active sessions
- Hardened authentication to verify users are still active on every permission check
- Hardened AI response parsing and JSON validation
- Restricted admin pages to verified admin users
- Prevented XSS and script injection in exercise content, math workspace, and comment tooltips
- Secured AI provider API keys by moving them from URLs to request headers
- Blocked unsafe external URLs and local network addresses in AI provider configuration
